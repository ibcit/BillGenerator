-- Format of the CSV file --

A CSV file is a text based file that contains information in a comma-delimited format. Usually the first line of text displayed, contains the headers of the file. An example of headers that may be provided to the program are:

  • FULLNAME - Name of Consignee (Required)
  • COMPANY - Company name of the Consignee
  • ADDRESS1 - Address for delivery (Required)
  • ADDRESS2 - Additional address field
  • CITY - City name (Required)
  • STATE - State name (Required for US addresses)
  • ZIPCODE - Zip Code (Required for US addresses)
  • COUNTRY - Country name (Required)
  • DESCRIPTION - Field for notes
  • QTY - Field for quantity shipped
  • WEIGHT - Shipment Weight

The CSV file can be generated directly from the customer or the file can be created from an existing Excel file, simply choose File -> "Save as" from the Excel Menu and on the file type drop-down select CSV format and save.


The IBC Airwaybill consists of 3 main parts, Shipper Information, Consignee Information and Shipment Information. For the purposes of creating IBC Airwaybills with a CSV file, there are 2 types of data that is processed by the BillGenerator program, static and variable data.


Static data refers to information that will be the same on every airwaybill, like the Shipper's information, such as account number, Shipper's name, address, etc. In constrast, the Consignee's information is en example of Variable information, as it will be different on each airwaybill created. The Shipment's information is made up with data that can be Static or Variable. If you set the Shipment's weight for example, as a global parameter, then every airwaybill will have this value, otherwise, if declared properly, this value will be processed from the file. Essentially, every line of the CSV file is the Variable information of an IBC Airwaybill, and contians Consignee and Shipment information.
