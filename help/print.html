-- Print Airwaybills (Step by Step) --

In order to produce accurate IBC Airwaybills you will need:

  • IBC Shipper account number and information such as Shipper name, address, etc.
  • The attributes of the merchandise that is beign shipped, such as the weight, value, description, packaging, and definition (DOCS or APX)
  • The types of services requested by the Shipping customer, such as second day delivery.
  • The Excel or CSV file containing the consignee addresses. It is very important to browse the file for accuracy as we cannot process airwaybills with missing name, or address fields.


To begin printing Airwaybills:

  1. Required} Open the IBC Airwaybill program, you will be prompted to choose a database from
    • PACTRAK - for distributions that originate in the US.
    • PACTRAK.LHR - for distributions that originate in the UK.
    • TESTDB - for testing purposes.
  2. Required} Type in your IBC username and password.
  3. Required} Open the CSV file (File -> Open or CTRL+O).
    The file will load into the main program window. You may open multiple CSV files and the program will accomodate each file as a new tab. Only the current selected tab (File) will be used when creating Airwaybills.
  4. Required} Enter the shipper information (Edit -> Shipper or CTRL+R).
    This Information must be filled out correctly and corresponds to the IBC Customer that generated the Consignee list and is sending the shipments. It is very important to be accurate with the shipper's Account Number, Reference Number, Second Reference number if any, etc. The Shipper reference number can contain a value that is read directly from the file. You can choose to have such value placed in the middle of the reference string by using a wild card value or by default at the end of the string. There are three available wild card values ($A, $B, $C) for a maximum of 3 actual entries read from the input file. For example, if you needed to create a reference field that contained the Lot Value read from the file you would write a string like this on the value field: BULK $A, LOT A Where the $A is the value that is going to be read from the file. You may also repeat that same value later in the string by typing another $A wild card in another place on the reference string. By default the first value draged into the reference list when you prepare the output, will be reference to $A, the second to $B and the third to $C. If you request 3 entries in your reference string, make sure you pass 3 variables to the drag and drop list, or the print request will stop. If you request 2 variables and pass more than 2 on the drag and drop list, only the referenced values will actually be placed on the reference field. Also, by default the reference value(s) will be added at the end of the string if no wild card value is specified. You can now load and save your default shipper information into a parameter file. It is prefered to store this files in the default selected storage path for easy loading from other users. Simply click the "Load" or "Save" option from the Shipper Information Box.
  5. Enter Shipment's description (Edit -> Description or CTRL+D).
    This is where the static value for the description will be defined, otherwise leave blank.
  6. Enter the Shipment's Information (Edit -> Shipement Info or CTRL+M).
    This is where all the static values pertaining to the shipments will be entered, including port of origin, value, weight, Service Level, etc, will be entered. You must first activate the appropriate entry boxes by checking the desired check box next the desired text field(s) or drop-down box(es).
  7. Select Additional "Run time" options (Edit -> Run Time Settings or CTRL+U). In these section you will be able to select additional functions such as
    • DHL Routing - based on each of the 3 main IBC Stations MIA, NYC and LAX. If needed you must select the appropriate station from which DHL will pick up or route the shipments.
    • AWB Settings - such as Multilot printing is useful when you had Distributions with dynamic quantity fields and each item must have it's own copy of the airwaybill. The corresponding airwaybill will have the label 1 of TOTAL# requested in the CSV file.
      Here you will also be able to request that an output file be created which reports: the date and time the airwaybill is created, the airwaybill number and the consignee address.
      In here you will also be able to request that the final output files be created in a 4 per page "Label" format. You must have access to a special 4 per page label sheet to be able to request these.
    • Lastly you can select to use LBS or KILOS as the default weight qualifier.
  8. On the current tab select the desired amount of Airwaybills to be created or Select ALL (CTRL+A) and click on the Print Icon (CTRL+P) to begin the Print Process.
  9. You will be prompted to DRAG and DROP the Column headers of the selected CSV files, into the Airwaybill sections. For example, on an input file that contains the headers FIRST_NAME, and LAST_NAME, these headers would be DRAGGED into the column Name in the DRAG-N-DROP window in the proper order (FIRST_NAME first and LAST_NAME second). Here is a list of all of the DRAG-N-DROP columns
    • Name
    • Company
    • Address
    • City
    • State
    • Zip
    • Country
    • Phone
    • Qty
    • Weight
    • Reference
    • 2nd Reference
    • Destination
    • Description
    All of these headers represent static data that will be read from the file and filled into each airwaybill, and these configuration can be stored and loaded on distributions where the headers on the file provided don't change.
  10. The Program will reserve Airwaybill numbers for each of the rows in the file, produce the output file (if requested), store the data in the selected database, and if all the data is successfully stored, the program will pause and ask the user to proceed with the printout. The last step is to simply choose a printer, load the paper and print.
